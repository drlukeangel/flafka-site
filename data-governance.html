FAQ

Common Governance Questions

Does Flafka support SOC 2 compliance?

Yes. Flafka's compliance automation covers SOC 2, PCI-DSS, GDPR, and HIPAA. The Compliance Posture Dashboard maps your configuration against each framework's control requirements. The Evidence Bundle Export produces a one-click ZIP with audit logs, access reviews, DQ reports, and posture snapshots ready for auditor submission.

Read our SOC 2 for Kafka blog post →

How does Kafka DLP work?

Flafka uses a hybrid regex + ML pipeline to detect 13 PII types in Kafka message payloads. Detection runs at sub-microsecond latency with no perceptible throughput impact. In producer blocking mode, Flafka rejects messages containing PII before they enter Kafka topics. A self-training feedback loop improves accuracy over time. All detection runs on your own infrastructure — no data leaves your environment.

See the full DLP explainer →

Can I import data quality rules in bulk?

Yes. Flafka supports CSV batch import for up to 200 DQ rules in a single upload. Each row in the CSV defines the rule name, topic, field path, condition type, threshold value, severity level, and alert setting. This is useful for migrating rules from existing DQ systems or setting up governance for new pipelines at scale.
