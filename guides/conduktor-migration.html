Configuration Migration

Connection Profiles

If you have Conduktor connection profiles configured, recreating them in Flafka takes about 2 minutes per cluster:

  1. In Conduktor, note your Bootstrap Server, API Key, and API Secret for each connection
  2. In Flafka, open Settings → Connection Profiles
  3. Click "Add Profile" and enter the same credentials
  4. Test the connection — Flafka will verify connectivity and list available topics

Flafka connects via SASL/PLAIN + TLS, the same mechanism Conduktor uses for Confluent Cloud clusters.

Access Control Mapping

Conduktor uses role-based access with basic roles. Flafka extends this with resource-level RBAC using glob patterns:

  • Conduktor "Admin" → Flafka "Admin" role (full access)
  • Conduktor "Editor" → Flafka custom role with * resource pattern
  • Conduktor "Viewer" → Flafka "Viewer" role (read-only)
  • Per-team access → Flafka glob patterns like team-payments-* restrict access to specific resources

Gateway Policies

If you use Conduktor Gateway for proxy-level policies (virtual clusters, data masking at the wire level), note that Flafka complements Gateway — use Gateway for proxy-level policies and Flafka for management, monitoring, and compliance. The two tools can coexist in your stack.
