Schema Impact Analysis

Before you evolve a schema, see exactly what will be affected. Flafka maps every topic, consumer group, Flink SQL statement, and connector that depends on a schema — giving you a complete blast radius visualization.

  • Dependency graph across topics, consumers, and Flink jobs
  • Visual blast radius before any schema change
  • Compatibility checking with conflict detection
  • Schema diff viewer for version comparison
  • Supports Avro, JSON Schema, and Protobuf
Learn more

How blast radius visualization works

When you select a schema for evolution, Flafka scans all topics using that schema, then traces every consumer group, Flink SQL statement, and connector subscribed to those topics. The result is a visual dependency graph showing exactly what will be affected by your change.

Example: You want to remove the legacy_id field from your orders-value schema. Flafka shows you that 3 topics use this schema, 5 consumer groups read from those topics, and 2 Flink SQL statements join on legacy_id. You know exactly what to fix before deploying.

Flink SQL IDE + Cost Estimation

A Monaco-powered SQL editor built specifically for Confluent Flink SQL. Write, test, and manage Flink SQL statements with pre-submission cost estimates that prevent billing surprises.

  • Monaco editor with Flink SQL syntax highlighting
  • Multi-cell notebook-style workflow
  • CFU/hour cost estimation before submission
  • Budget warnings when estimates exceed limits
  • Statement lifecycle: create, monitor, pause, delete
  • SQL template library with reusable patterns
Learn more

Cost estimation workflow

Before submitting any Flink SQL statement, Flafka estimates the CFU/hour cost based on the query complexity, input topic throughput, and windowing operations. If the estimate exceeds your configured budget threshold, a warning appears with the projected monthly cost.

Example: A SELECT ... FROM orders GROUP BY TUMBLE(event_time, INTERVAL '5' MINUTE) query estimates at 2.4 CFU/hour ($57.60/month). Your budget limit is $50/month — Flafka warns you before submission.

Full lifecycle management plus observability that no other Kafka GUI provides. Throughput time-series charts show exactly how your connectors are performing, and structured logs make debugging instant.

Throughput charts and log search

Throughput charts display messages/sec and bytes/sec as time-series data, making it easy to spot anomalies — a sudden drop to zero, a throughput spike, or a gradual degradation. The log viewer provides structured, searchable connector logs with severity filtering (INFO, WARN, ERROR) and full-text search.

Example: Your S3 sink connector's throughput drops to zero at 3:14am. You search the logs for "ERROR" and find a permission denied error from AWS IAM. Root cause identified in seconds, not hours.

Resource-Level RBAC

Fine-grained access control using glob patterns. Assign permissions like analytics-* or prod.orders.* to control exactly who can access which resources. PCI-DSS Requirement 7 aligned.

  • Glob pattern matching for resource-level access
  • ACL template presets (Consumer, Producer, Admin)
  • Bootstrap admin with lockout prevention
  • Permission-denied guidance for users
  • PCI-DSS Requirement 7 alignment
  • Full audit trail for compliance
Learn more

Glob pattern examples

Resource patterns use glob syntax familiar to any developer. Examples:

  • analytics-* — All analytics topics
  • prod.* — All production resources
  • team-payments-* — Payment team's resources

PCI-DSS Requirement 7 mandates that access to cardholder data is restricted to need-to-know. Flafka's resource-level RBAC with audit trails provides the access control and evidence trail that auditors require.

SLO Tracking + Error Budgets

Define service level objectives for your Kafka infrastructure. Track lag, throughput, and availability against your targets with error budget burn-down visualizations that keep you ahead of incidents.

  • Custom SLOs for lag, throughput, and availability
  • Error budget burn-down visualization
  • 7-day sparkline trends
  • Executive KPI dashboard
  • Health monitoring integration
  • Telemetry-driven metrics
Learn more

Error budget calculation and burn-down

Define SLOs like "99.5% availability for orders topic" or "P99 consumer lag under 500ms." Flafka calculates your error budget — the amount of failure you can tolerate within your SLO window — and displays a burn-down visualization showing how quickly you're consuming it.

When your error budget burns faster than expected, you see it in real-time. 7-day sparkline trends on the dashboard show whether your reliability is improving or degrading, giving you time to act before an SLO breach.
