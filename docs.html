Zero to running in 5 minutes

Get Started with Flafka

Everything you need to install, configure, and start managing your Kafka clusters. Unlike Conduktor (requires Docker Compose + Kafka Connect), Flafka runs with a single Docker command and zero configuration files.

Quick Start with Docker

Launch Flafka with a single command. No configuration files, no dependencies.

bash 
docker run -p 8080:8080 flafka/flafka:latest

Then open http://localhost:8080 in your browser. That's it — Flafka serves both the web UI and API from a single container.

Supported Platforms

Flafka currently supports Confluent Cloud with full feature coverage (Kafka, Flink SQL, ksqlDB, Schema Registry). Support for AWS MSK and self-hosted Apache Kafka is planned.

Configuration

All configuration is done through environment variables. Pass them via docker run -e or a .env file.

Variable Type Default Description
FLAFKA_CONFLUENT_API_KEY string — Confluent Cloud API key for authentication
FLAFKA_CONFLUENT_API_SECRET string — Confluent Cloud API secret for authentication
FLAFKA_CONFLUENT_BOOTSTRAP_SERVER string — Confluent Cloud bootstrap server URL
FLAFKA_CONFLUENT_CLUSTER_ID string — Confluent Cloud cluster identifier
FLAFKA_CONFLUENT_ENV_ID string — Confluent Cloud environment identifier
FLAFKA_SERVER_PORT number 8080 Port for the Flafka web server and API
FLAFKA_SERVER_HOST string 0.0.0.0 Bind address for the server
FLAFKA_SPA_DIR path /app/dist Directory containing the built React frontend assets
FLAFKA_LOG_LEVEL string info Logging level: trace, debug, info, warn, error
bash 
docker run -p 8080:8080 \
  -e FLAFKA_CONFLUENT_API_KEY=your-api-key \
  -e FLAFKA_CONFLUENT_API_SECRET=your-api-secret \
  -e FLAFKA_CONFLUENT_BOOTSTRAP_SERVER=pkc-xxxxx.us-east-1.aws.confluent.cloud:9092 \
  flafka/flafka:latest

First Flink SQL Statement

Run a streaming query on live Kafka data in under 2 minutes.

  1. 1

    Navigate to the Workspace

    From the sidebar, click Workspace to open the SQL editor.

  2. 2

    Create a Source Table

    CREATE TABLE orders (
  order_id STRING,
  amount DECIMAL(10,2),
  ts TIMESTAMP(3),
  WATERMARK FOR ts AS ts - INTERVAL '5' SECOND
) WITH (
  'connector' = 'kafka',
  'topic' = 'orders',
  'properties.bootstrap.servers' = 'YOUR_BOOTSTRAP_SERVERS',
  'format' = 'json'
);
  3. 3

    Run a Streaming Query

    SELECT
  TUMBLE_START(ts, INTERVAL '1' MINUTE) AS window_start,
  SUM(amount) AS revenue
FROM orders
GROUP BY TUMBLE(ts, INTERVAL '1' MINUTE);
  4. 4

    View Live Results

    Results stream to the output panel in real time. Click Stop to end the job.

First Steps

Go from zero to monitoring in 5 minutes.

1

Launch Flafka (~30 seconds)

Run the Docker one-liner above, or launch the desktop app. Open http://localhost:8080 to see the Flafka dashboard.

2

Add Confluent Cloud Credentials (~1 minute)

Enter your Confluent Cloud API key, secret, and bootstrap server. Flafka connects automatically via SASL/TLS — no additional TLS configuration needed.

3

Browse Topics (~30 seconds)

View all your Kafka topics with real-time message counts, partition details, schema information, and consumer group lag. Click any topic to browse messages live.

4

Write Flink SQL (~2 minutes)

Open the SQL editor with built-in autocomplete, syntax highlighting, and statement lifecycle management. Write and execute Flink SQL statements against your streams.

5

Set Up SLO Monitoring (~1 minute)

Configure service level objectives for consumer lag, throughput, and error rates. Get real-time dashboards and alerts when SLOs are at risk.

Architecture Overview

Flafka is a single binary that serves both the backend API and the frontend UI. No microservices, no message brokers, no external databases.

Troubleshooting

Port 8080 is already in use

Map to a different port: docker run -p 3000:8080 flafka/flafka:latest

Connection refused to Confluent Cloud

Verify your API key and bootstrap server URL. Ensure port 9092 is not blocked by your firewall.

Invalid API key format

Confluent Cloud API keys are alphanumeric strings. Do not include quotes around values in -e flags.

Docker permission denied

On Linux, add your user to the docker group: sudo usermod -aG docker $USER, then log out and back in.
